Отчет по ознакомительной практике: Применение полученных теоретических знаний в области информационных технологий, развитие профессиональных навыков, а также получение опыта работы в реальном проекте. 


#1. Информация о датасете
Для выполнения работы использовался датасет №1 https://www.kaggle.com/datasets/sanjeevsahu/web-log-access-dataset 


#2. Шаги установки и настройки
Развертывание системы производилось в Docker-контейнерах.

Установил Docker Desktop

Для хранения данных индексов OpenSearch было настроено монтирование директории openserch_data на хост-машину.

Развернул и связал между собой OpenSearch и OpenSearch Dashboards.

Был подготовлен конфигурационный файл "logstash.conf". Данные считывались из файла логов с помощью плагина File Input Plugin. 
Парсинг логов реализован через плагин Grok Filter Plugin с выделением полей:
\* "clientip" (IP-адрес)
\* "timestamp" (время события)
\* "http.request.method" (метод запроса)
\* "url.original" (запрашиваемый ресурс)
\* "http.response.status_code" (код ответа)
\* "bytes" (объем переданных данных)

Очищенные и структурированные данные отправлялись в индекс OpenSearch через opensearch output плагин.

Открыт в браузере OpenSearch Dashboards.

В OpenSearch Dashboards был собран дашборд из 6 диаграмм.


#3. Сложности в процессе выполнения работы 

1. При запуске Docker Desktop была ошибка невозможности обновления wsl. Решилась переключением запуска контейнеров на HyperV.
2. Проблема с авторизацией в OpenSearch Dashboards решилась отключением авторизации.
3. При запуске OpenSearch каждый раз падал из-за недостатка памяти, это решилось переводом OpenSearch на использование стандартного системного хранилища.
4. Из-за слишком маленького масштаба временного фильтра в OpenSearch Dashboards при выводе графиков на них отсутствовала инфирмация. Решилось установкой большего времяни.
